## ChatGPT o1

Right now, Bedrock‚Äôs Anthropic model interface **does not** support a separate "system" role in the messages array. Instead, Anthropic‚Äôs ‚Äúsystem‚Äù instructions are meant to go in the **`context`** field at the top level of your JSON body. Only `"user"` and `"assistant"` are valid values for the `role` key in `messages`.

That means you have two main changes to make:

1. **Remove** the `{ role: "system", content: ... }` entry from your `messages` array.  
2. **Add** your system prompt as the `context` property in the JSON you pass to Bedrock.

For example:

```js
// Instead of:
const payload = {
  anthropic_version: "bedrock-2023-05-31",
  max_tokens: 10000,
  messages: [
    { role: "system", content: systemPrompt },  // <-- remove this
    { role: "user", content: "Hello there." },
    { role: "assistant", content: "Hi, I'm Claude. How can I help you?" },
    { role: "user", content: "Can you explain LLMs in plain English?" },
  ],
};

// Use ‚Äúcontext‚Äù for system instructions:
const payload = {
  anthropic_version: "bedrock-2023-05-31",
  max_tokens: 10000,

  // Put the system prompt here:
  context: systemPrompt,

  // Only user/assistant roles in messages
  messages: [
    { role: "user", content: "Hello there." },
    { role: "assistant", content: "Hi, I'm Claude. How can I help you?" },
    { role: "user", content: "Can you explain LLMs in plain English?" },
  ],
};
```

Then invoke the model as usual:

```js
const command = new InvokeModelWithResponseStreamCommand({
  contentType: "application/json",
  body: JSON.stringify(payload),
  modelId,
});
```

This ensures Bedrock won‚Äôt reject the request with the ‚Äúrole: system is not valid‚Äù error, and Claude will apply your system prompt (now provided in `context`). This approach aligns with Anthropic‚Äôs ‚Äúsystem prompt‚Äù concept, but in Bedrock‚Äôs API format, it goes under `context` rather than as a separate system role.
